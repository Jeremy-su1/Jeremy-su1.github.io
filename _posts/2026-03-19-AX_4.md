### RRF (Reciprocal Rank Fusion)
- 여러 검색 결과를 하나로 통합하는 순위 결합 알고리즘

- 작동 방식:
1. 각 검색 결과의 순위를 역수로 변환
2. 상수 k를 더해 안정성 확보
3. 모든 결과의 RRF 점수를 합산

- langchain emsemble retriever == RRF 방식을 채용

```
from langchain.schema import StrOutputParser
from langchain.schema.runnable import RunnablePassthrough
from langchain.prompts import ChatPromptTemplate

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

# 프롬프트 템플릿 정의
template = """
당신은 금융 상품을 고객에게 소개하는 전문가입니다.
아래는 질문에 답변하는데 도움이 되는 문서들입니다:

{context}

위 문서들을 참고하여 다음 질문에 답변해주세요:
{question}

답변은 친절하고 이해하기 쉽게 작성해주세요."""

prompt = ChatPromptTemplate.from_template(template)

# LCEL Chain 구성
semantic_retriever_chain = (
        {
            "context": semantic_retriever | format_docs,
            "question": RunnablePassthrough()
        } # 입력값을 프롬프트의 {context}와 {question}에 매핑, RunnablePassthrough()은 질문을 그대로 전달. 이 두개를 병렬 처리
        | prompt
        | llm
        | StrOutputParser()
)
```

```
  🎯 RAGAS란?                                                                                                                                        
                                                                                                                                                       
  RAGAS는 RAG (Retrieval-Augmented Generation) 시스템을 평가하기 위한 프레임워크입니다.                                                                
                                                                                                                                                       
  핵심 개념                                                                                                                                            
                  
  1. LLM as a Judge 패러다임
  - 기본적으로 OpenAI의 gpt-4o-mini를 사용하여 RAG 시스템을 평가
  - 인간 평가자의 주관성과 편향을 줄이고 일관된 평가 기준 적용
  - 수작업 평가의 시간/비용 절감

  2. 주요 평가 지표

  ┌────────────────────┬─────────────────────────────────────────┐
  │        지표        │                  의미                   │
  ├────────────────────┼─────────────────────────────────────────┤
  │ Context Precision  │ 검색된 문서가 질의와 얼마나 관련있는가? │
  ├────────────────────┼─────────────────────────────────────────┤
  │ Context Recall     │ 필요한 정보를 모두 포함하고 있는가?     │
  ├────────────────────┼─────────────────────────────────────────┤
  │ Faithfulness       │ 응답이 문맥(컨텍스트)에 충실한가?       │
  ├────────────────────┼─────────────────────────────────────────┤
  │ Answer Relevancy   │ 질문에 대한 답변이 적절한가?            │
  ├────────────────────┼─────────────────────────────────────────┤
  │ Answer Correctness │ 정답과 비교했을 때 얼마나 정확한가?     │
  └────────────────────┴─────────────────────────────────────────┘

  3. 평가 대상 요소
  - 🔹 사용자 질의 (Query)
  - 🔹 검색된 컨텍스트 (Context)
  - 🔹 LLM이 생성한 응답 (Response)
  - 🔹 기준 정답 (Ground Truth) (선택적)

  특징

  ✅ 자동화된 평가: 대규모 데이터에 대해 일관된 평가 기준 적용
  ✅ 유연성: Amazon Bedrock, Claude, LLaMA 등 다양한 LLM 사용 가능
  ✅ 확장성: 병렬 처리로 대량 평가 수행
  ✅ 커스터마이징: AspectCritic, RubricsScore, Custom Metric 등 커스텀 메트릭 생성 가능

  실제 사용 예시

  from ragas.metrics import LLMContextPrecisionWithoutReference
  from ragas import SingleTurnSample

  # RAGAS 메트릭 초기화
  context_precision = LLMContextPrecisionWithoutReference(llm=wrapped_openai_llm)

  # 평가 샘플
  sample = SingleTurnSample(
      user_input="군인 예적금 상품의 이자율은?",
      response="연 4.5%입니다",
      retrieved_contexts=["...검색된 문서들..."]
  )

  # 평가 실행
  score = await context_precision.single_turn_ascore(sample)

  📌 요약: RAGAS = RAG 시스템의 품질을 객관적으로 측정하는 자동화된 평가 도구
```
